## 📘 HTML Boilerplate — Why Each Part Matters

Before diving into the exercises, I learned the purpose of the standard HTML5 boilerplate. Each part plays a specific role:

### **`<!DOCTYPE html>`**
Tells the browser that the document uses HTML5. This prevents the browser from switching into “quirks mode” and ensures consistent rendering.

### **`<html>`**
The root element that wraps the entire webpage. Everything inside the document lives here.

### **`<head>`**
Contains metadata *about* the page — things the user doesn’t see directly, like the title, character encoding, and viewport settings.

### **`<meta charset="UTF-8">`**
Ensures the page supports modern characters, symbols, and emojis. Without this, special characters may break.

### **`<title>`**
Controls the text shown in the browser tab and in bookmarks. It’s the official name of the page.

### **`<body>`**
Everything visible to the user — text, images, links, lists, and all page content — goes inside the body.
